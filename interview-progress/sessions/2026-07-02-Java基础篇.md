## Q54 编译+解释/JIT？

**我的回答要点**：Java 先编译，计数器到一定值保存到本地，之后再执行就是解释了；Git 是计数器，达到数值就保存到本地

**得分**：16/40 | 较差

**纠错**：
- 方向反了：不是「编译后变解释」，而是 **启动时先解释执行**，热点代码再由 **JIT 编译成本地机器码**
- JIT = Just-In-Time，不是 Git；方法调用计数器触发的是 **JIT 编译**，不是「保存后改解释」
- 完整链路：`javac` 编译成字节码 → JVM 解释器逐条执行 → 热点达阈值 → JIT 编译机器码并缓存 → 后续直接跑机器码

---

## Q55 编译型 vs 解释型？

**我的回答要点**：解释型运行时逐行解释执行；编译型运行前就编译好直接运行；Java 属于先解释再编译

**得分**：26/40 | 一般

**纠错**：
- 基本区分 ✅，但 Java 不能只说「先解释再编译」
- 完整说法：**半编译半解释** — `javac` 先编译成**字节码**（编译期），JVM 运行时**解释 + JIT**（运行期）
- 对比：C/C++ 直接编译成机器码；Python 传统上偏解释（也有字节码）
