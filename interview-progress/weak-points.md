# 薄弱点汇总

> 复习优先级：🔴 高 → 🟡 中 → 🟢 低

## 🔴 高优先级（追问必挂）

### 1. 泛型与泛型擦除
- **问题**：曾与多态混淆；不知道擦除是什么
- **要记住**：
  - 泛型 = 编译期类型检查，不是多态
  - 擦除 = 编译后 `T` 消失，运行时看不到具体类型
  - 不能 `new T()` 因为运行时不知道 T 是谁
- **一句话**：多态运行时决定调谁的方法；泛型编译时决定能放什么类型

### 2. final 与不可变类
- **问题**：以为 `final` 类 = 对象不可变
- **要记住**：
  - `final` 类 = 不能被继承
  - 不可变类还需：字段 private final、无 setter、可变对象要防御性拷贝
- **例子**：`final class User { StringBuilder name; }` 仍可变

### 3. 编译期 vs 运行时
- **问题**：一直不理解「编译器」是什么意思
- **要记住**：
  - 编译器（javac）= 程序运行前检查+翻译
  - JVM = 程序运行时执行
  - 泛型检查、受检异常、方法重载 → 编译期
  - 多态、NPE、数组越界 → 运行时

## 🟡 中优先级（知道但不严谨）

### 4. String 不可变
- 知道结论，但说不清常量池、hash 缓存、安全性
- `final` 不等于不可变，要分开讲

### 5. equals 与 hashCode
- 知道要一起重写，但说过「hashCode 返回 true」（应为 int，且是「相等」）
- `Object.equals()` 默认等价于 `==`

### 6. 反射
- 知道能动态调用，但场景不全，缺点没讲
- Spring 是 **IoC** 不是 iop

### 7. 抽象类 vs 接口
- 大方向对，但 Java 8 的 default/static 方法不熟
- 「接口方法必须重写」不严谨

### 8. 异常体系
- 知道 Error/Exception，但受检/非受检划分依据说不清
- 受检异常也是运行时抛出，不是「编译时异常」

## 🟢 已掌握（可少花时间）

- Integer 缓存池（-128~127）
- 深拷贝 vs 浅拷贝核心区别
- == 与 equals 基本场景

## 易混概念对照

| 概念 A | 概念 B | 区分 |
|---|---|---|
| 多态 | 泛型 | 运行时行为 vs 编译期类型检查 |
| final 类 | 不可变类 | 不能继承 vs 状态不能变 |
| 受检异常 | 非受检异常 | 是否继承 RuntimeException |
| 方法重载 | 方法重写 | 编译期 vs 运行时 |
| 浅拷贝 | 深拷贝 | 引用共享 vs 递归新建 |
| BIO | NIO | 待学习 ← 下一题 |
