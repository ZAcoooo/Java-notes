# 薄弱点汇总

> 复习优先级：🔴 高 → 🟡 中 → 🟢 低

## 🔴 高优先级（追问必挂）

### 1. 泛型与泛型擦除
- **问题**：曾与多态混淆；Q17 有进步但仍说「运行时擦除」
- **要记住**：
  - 泛型 = 编译期类型检查，不是多态
  - 擦除 = **javac 编译后** `T` 消失，运行时当原始类型处理
  - 不能 `new T()` 因为编译后根本没有 T 的类型信息
  - 需要类型时用 `Class<T>` 传入
- **一句话**：多态运行时决定调谁的方法；泛型编译时决定能放什么类型

### 2. final 与不可变类
- **问题**：以为 `final` 类 = 对象不可变
- **要记住**：
  - `final` 类 = 不能被继承
  - 不可变类还需：字段 private final、无 setter、可变对象要防御性拷贝
- **例子**：`final class User { StringBuilder name; }` 仍可变

### 3. 编译期 vs 运行时
- **问题**：一直不理解「编译器」是什么意思
- **要记住**：
  - 编译器（javac）= 程序运行前检查+翻译
  - JVM = 程序运行时执行
  - 泛型检查、受检异常、方法重载 → 编译期
  - 多态、**方法重写**、NPE、数组越界 → 运行时
- **Q18 进步**：重载/重写与编译期/运行时已能正确区分 ✅

### 9. Stream API
- **问题**：不知道中间操作 vs 终端操作；stream 理解成「单线程」
- **要记住**：
  - 懒执行原因：**等终端操作触发才真正执行**，便于优化（短路、合并操作），不是因为「能链式调用」
  - 终端操作（collect/count/forEach）触发真正计算，返回结果
  - `stream()` = 顺序流；`parallelStream()` = 并行流，适合 CPU 密集 + 大数据量
- **易错**：I/O 密集不适合 parallelStream；没有终端操作中间操作不执行

### 10. 内部类
- **问题**：混淆创建方式；「静态内部类无法被实例对象修改」表述不清
- **要记住**：
  - 非静态内部类 **依赖外部类实例**，隐式持有外部类 `this` 引用
  - 静态内部类 **不依赖**外部类实例，类似静态成员
  - 创建：`outer.new Inner()` vs `new Outer.StaticInner()`
  - 访问：非静态可访问外部实例成员；静态只能访问外部 static 成员

### 11. JVM 内存区域（静态 vs 实例变量）
- **问题**：知道共享/独享区别，不知道存哪
- **要记住**：
  - 静态变量 → **方法区**（JDK 8+ 叫 **元空间 Metaspace**）
  - 实例变量 → **堆 Heap**（随对象一起分配）
  - 静态变量类加载时初始化，只有一份

### 12. BigDecimal
- **问题**：Q22 仍用 `new BigDecimal(0.1)` — **这是面试第一陷阱！**
- **要记住**：
  - `0.1 + 0.2` = `0.30000000000000004`（背完整）
  - ✅ `new BigDecimal("0.1")` 或 `BigDecimal.valueOf(0.1)`
  - ❌ `new BigDecimal(0.1)` — 0.1 先变成不精确的 double 再传入
  - 运算：`bd1.add(bd2)` 返回新对象，不能写 `bd1 + bd2`

### 13. 类加载/初始化顺序
- **问题**：把父类构造放在子类 static 之前；漏掉实例代码块
- **要记住**（`new Son()` 完整 6 步）：
  1. 父类 static（变量+静态块，按代码顺序）
  2. 子类 static（变量+静态块）
  3. 父类实例变量+实例代码块
  4. 父类构造方法
  5. 子类实例变量+实例代码块
  6. 子类构造方法
- **口诀**：静态只一次，先父后子；构造先父后子

## 🟡 中优先级（知道但不严谨）

### 4. String 不可变
- 知道结论，但说不清常量池、hash 缓存、安全性
- `final` 不等于不可变，要分开讲

### 5. equals 与 hashCode
- 知道要一起重写，但说过「hashCode 返回 true」（应为 int，且是「相等」）
- `Object.equals()` 默认等价于 `==`

### 6. 反射
- 知道能动态调用，但场景不全，缺点没讲
- Spring 是 **IoC** 不是 iop

### 7. 抽象类 vs 接口
- 大方向对，但 Java 8 的 default/static 方法不熟
- 「接口方法必须重写」不严谨

### 8. 异常体系
- 知道 Error/Exception，但受检/非受检划分依据说不清
- 受检异常也是运行时抛出，不是「编译时异常」

## 易混概念对照

| 概念 A | 概念 B | 区分 |
|---|---|---|
| 多态 | 泛型 | 运行时行为 vs 编译期类型检查 |
| final 类 | 不可变类 | 不能继承 vs 状态不能变 |
| 受检异常 | 非受检异常 | 是否继承 RuntimeException |
| 方法重载 | 方法重写 | 编译期 vs 运行时 |
| 浅拷贝 | 深拷贝 | 引用共享 vs 递归新建 |
| BIO | NIO | 同步阻塞一线程一连接 vs 同步非阻塞多路复用 |
| NIO | AIO | NIO 同步非阻塞自己处理 vs AIO 异步回调 |
